# 技能、工作区与自动化

当基础聊天和模型都跑通之后，Omnibot 的价值主要体现在“会做事”。这一页把技能、工作区、定时任务和 MCP 串起来。

## 1. 先打开技能商店

技能商店页支持：

- 搜索技能
- 安装内置技能
- 启用 / 禁用
- 删除技能

推荐先装那些不会直接影响系统稳定性的轻量技能，验证安装链路无误后，再逐步开启更强的自动化能力。

## 2. 使用 Workspace

工作区页当前已经支持：

- 浏览目录
- 返回上级目录
- 打开 Artifact 预览
- 与聊天和工具结果联动

适合验证的几个动作：

- 打开工作区根目录
- 查看 `.omnibot/` 下的配置与记忆文件
- 预览某个生成物或附件

![Workspace 浏览](/assets/workspace.jpg)

## 3. 让 Agent 学会处理定时任务

项目把“提醒”和“执行”分成两类：

- Alarm：提醒
- Scheduled task：在未来某个时间真正执行任务

一个简单实操路径是：

1. 创建一个定时任务
2. 设定执行时间
3. 观察任务是否被恢复与触发

## 4. 打开浏览器与工作区联动能力

聊天页里已经存在浏览器叠层和文件选择链路，适合做这类验证：

- 让 Agent 打开某个网页
- 查看 Browser Mirror 是否跟随
- 验证文件上传 / 选择是否可用

![浏览器能力](/assets/browser.jpg)

## 5. 启用本地 MCP 服务

如果你希望别的设备或桌面页面接入这台手机上的 Omnibot，可以在设置里开启本地 MCP。

开启后优先验证：

- 健康检查地址是否可达
- `webchat/` 是否能打开
- Token 是否生成

这一步成功后，Omnibot 就不再只是手机上的一个 UI，而是局域网里的可访问 Agent 节点。

## 6. 配置远端 MCP 工具

远端 MCP 工具页适合接入外部服务能力，页面内可直接：

- 新增服务
- 编辑配置
- 启用 / 停用
- 刷新工具列表

建议做法：

1. 先只接一个最简单的远端 MCP 服务
2. 刷新工具列表确认协议正常
3. 再让 Agent 在聊天中尝试调用

## 7. 一条推荐的进阶路径

为了避免排查时相互影响，建议按这个顺序逐步打开能力：

1. 先跑通聊天与模型
2. 再补齐权限
3. 再开技能商店
4. 再验证工作区
5. 再尝试定时任务
6. 最后启用本地 / 远端 MCP

这样每一步都有清晰的成功标准。
